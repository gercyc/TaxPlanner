# Brainstorms — TaxPlanner

Saídas brutas de discovery e ideação. Cada brainstorm captura o
entendimento inicial de um problema, hipóteses, alternativas e decisões
pendentes **antes** de virar um plano formal.

## Como conduzir um brainstorm

1. Criar arquivo em `docs/brainstorms/AAAA-MM-DD-<slug-curto>.md`.
2. Seguir o esqueleto abaixo.
3. Ao final, listar **decisões abertas** e **próximos passos** (mover para
   `docs/plans/` quando virar execução).

## Esqueleto

```md
# Brainstorm — Título

- **Data**: AAAA-MM-DD
- **Autor(es)**: ...
- **Status**: Rascunho | Em revisão | Consolidado

## Problema
...

## Hipóteses
- ...

## Alternativas consideradas
- ...

## Riscos e dúvidas
- ...

## Próximos passos
- [ ] ...
```
